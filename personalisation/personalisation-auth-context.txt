1. THE USER FLOW
================================================================================

  Landing Page (index.tsx)
      ↓  User clicks "Launch Workspace" or "Launch App"
  YOUR AUTH MODAL (Google OAuth via Supabase)
      ↓  On successful auth → you receive a Supabase session/token
  REDIRECT TO /onboarding (our personalisation flow)
      ↓  User walks through 5 preference selection screens
  REDIRECT TO /dashboard (main app)

================================================================================
2. WHAT WE NEED FROM AUTH
================================================================================

After successful Google Auth, we need you to:

  a) Store the Supabase session so it's accessible app-wide.
     We recommend using a React context or Zustand store.
     The supabase client is already configured at:
       → src/lib/supabase.ts

  b) Navigate to "/onboarding" on first login (no profile yet)
     or to "/dashboard" if the user already has a profile.

     How to check: Query the `user_profiles` table in Supabase:
       const { data } = await supabase
         .from('user_profiles')
         .select('id')
         .eq('user_id', session.user.id)
         .single();
       
       if (data) navigate to /dashboard
       else navigate to /onboarding

  c) Make the authenticated user's ID available so we can save
     their personalisation preferences. We need:
       - user.id (Supabase UUID)
       - user.email (for display)
       - user.user_metadata.full_name (for greeting)
       - user.user_metadata.avatar_url (for profile pic)

================================================================================
3. SUPABASE TABLE: user_profiles
================================================================================

We will create this table. You do NOT need to create it. But here's the
schema so you know what we're storing:

  CREATE TABLE user_profiles (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id         UUID REFERENCES auth.users(id) ON DELETE CASCADE,
    cognitive_profile    TEXT,          -- 'adhd' | 'autism' | 'audhd' | 'high_load' | 'prefer_not_to_say'
    communication_style  TEXT,          -- 'checklists' | 'bullet_points' | 'short_paragraphs' | 'visual_kanban'
    peak_focus_time      TEXT,          -- 'morning' | 'afternoon' | 'evening' | 'variable'
    working_hours_start  TEXT,          -- e.g. '09:00'
    working_hours_end    TEXT,          -- e.g. '18:00'
    stress_triggers      TEXT[],        -- array of trigger strings
    urgency_preference   TEXT,          -- 'assume_urgent' | 'use_context' | 'ask_clarification' | 'default_low'
    onboarding_completed BOOLEAN DEFAULT FALSE,
    created_at           TIMESTAMPTZ DEFAULT NOW(),
    updated_at           TIMESTAMPTZ DEFAULT NOW()
  );

The `onboarding_completed` flag is what you should check to decide
whether to route to /onboarding or /dashboard after auth.
