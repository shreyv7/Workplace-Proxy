WORKPLACE PROXY - ROLE 1 (FULL-STACK & INTEGRATION LEAD) TOOLKIT

================================================================================
1. FRONTEND & CLIENT STACK
================================================================================
- React 19 & Vite 8: Project runtime.
- TanStack Start & Router: For file-based routing and lightning-fast server/client rendering.
- Tailwind CSS v4: Implementing the premium, neurodivergent-friendly Cognitive OS theme.
- Lucide React: System iconography.

================================================================================
2. DATABASE & REAL-TIME SYNC (SUPABASE)
================================================================================
- Supabase JS SDK (@supabase/supabase-js): Direct querying, updates, and inserts.
- Supabase Realtime (WebSockets):
  Used to subscribe to PostgreSQL database changes in real time. When Role 2's Python 
  backend updates the status of a message or logs a trace, the frontend UI updates 
  immediately without a page reload.

================================================================================
3. MODEL CONTEXT PROTOCOL (MCP) BRIDGES
================================================================================
- MCP TypeScript SDK (@modelcontextprotocol/sdk): Used to build Node.js MCP servers.
- Mock Environment Servers:
  * Slack MCP Bridge: Simulates reading raw inbound message streams from Slack channels.
  * Google Calendar MCP Bridge: Interacts with the user's mock calendar to block out slots.
  * Gmail MCP Bridge: Simulates reading incoming emails.

================================================================================
4. SHARED API & CONTRACT BRIDGE
================================================================================
- HTTP/Fetch API: Queries Role 2's Python backend (FastAPI).
- Contract Payload:
  - Inbound Message JSON sent to Python backend:
    {
      "source": "slack" | "email" | "jira" | "teams",
      "sender": "Sender Name",
      "text": "Raw unstructured text payload"
    }
  - Debate resolving results written directly to Supabase by the agents.

================================================================================
5. DEPLOYMENT AND HOSTING
================================================================================
- Vercel CLI: Deploying the TanStack Start frontend app.
- Supabase Dashboard: Designing public SQL schemas (messages, calendar_blocks, trace_logs).
