Role 1: The Full-Stack & Integration Lead
Your Mission: Build the user interface and establish the real-world connections. You own the
mock environments and how the user interacts with the final product.
● Vibe-Code the Frontend: Use Lovable.dev and your preferred agentic workflows to
rapidly generate the minimal "side-by-side" dashboard and daily calendar view.
● Deploy the App: Push the frontend to Vercel and set up a Supabase instance if you need
simple relational data tracking alongside the vector database.
● Build the MCP Bridges: Write the Model Context Protocol (MCP) servers that connect
the mock Slack/Email environments and Google Calendar to the backend. You are
responsible for ensuring data flows cleanly in and out of the application.
Role 2: The Multi-Agent Orchestrator
Their Mission: Build the "brain" of Workplace Proxy. This person works entirely in Python,
focusing on prompt engineering and the Agent-to-Agent (A2A) debate architecture.
● Configure Lyzr & Gemini: Initialize the Lyzr framework using the Google Agent
Development Kit (ADK) and Gemini models.
● Develop the Swarm: Code the individual personas: The Interceptor, The Contextualizer,
The Scheduler, and The Translator.
● Orchestrate the Debate: Write the logic that allows these agents to pass context back
and forth, debate the urgency of a message, and finalize the output before sending it to
the frontend.
Role 3: The Memory & Infrastructure Lead
Their Mission: Handle the data. This person ensures the agents have instant, accurate access
to the user's cognitive preferences and corporate jargon.
● Self-Host Qdrant: Set up the Docker container for Qdrant and ensure it is running
smoothly for local development.
● Build the RAG Pipeline: Write the Python scripts to chunk, embed, and store mock
corporate data (like old Slack threads or company policy docs) into Qdrant.
● Manage Context Retrieval: Create the specific search functions that the Multi-Agent
Orchestrator will call to fetch user preferences and project context during the A2A debate.

The Integration Plan
To make this work independently, you need a single contract between the three of you: a shared
API schema. Role 1 (shrey) will send a JSON payload (the raw Slack message) to a FastAPI
endpoint. Role 2 and Role 3 will process that message through Lyzr and Qdrant, and return a
clean JSON payload (the translated task and calendar slot) back to your frontend.
